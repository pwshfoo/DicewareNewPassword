About Diceware

Diceware is a method for generating strong passphrases by simulating dice rolls using a public wordlist. Each sequence of five dice rolls maps to one word from the EFF's large wordlist of 7,776 carefully selected English words.

Why Diceware?

  • Memorable: Natural language words are easier to remember than random characters
  • Strong entropy: Each word contributes ~12.9 bits; 4 words + 5 salt chars ≈ 87 bits (exceeds 80-bit "Tier 0" standard)
  • Auditable: Save the dice rolls and reconstruct your passphrase exactly
  • Public method: The wordlist and algorithm are fully documented — security comes from the randomness, not obscurity

Security & Entropy

Default passphrase (4 words + 5 salt chars): ~86.9 bits of entropy

✓ Meets Enterprise Requirements

Tier 0 / Domain Admin: ≥ 80 bits required — this exceeds it by 6.9 bits. Use -WordCount 5 for ~100 bits.

Entropy breakdown:

  • 4 Diceware words: 51.7 bits
  • 5 salt characters (95 unique): 32.9 bits
  • Random salt position: +2.3 bits

Why the public wordlist doesn't weaken it: Diceware follows Kerckhoffs's principle — security comes from the secret (your dice rolls), not obscurity. AES, RSA, and SHA-256 are all public too.

PowerShell Module

This project includes a revamped NewPassword.psm1 module — a complete rewrite of a legacy module used across many production scripts. It generates cryptographically secure passphrases, random character strings, and supports AES-256-GCM encryption with optional LSB steganography for offline password sharing.

Installation

Import-Module .\NewPassword\NewPassword.psm1

Quick Start

Generate a passphrase:

New-Password -WordCount 4 -SaltChars 5 -UppercaseFirstLetter

Export encrypted + embedded in stego images:

New-Password -Export -KeyCarrierImage .\photo1.jpg -DataCarrierImage .\photo2.jpg

Recover from encrypted files:

Get-Password -Import -KeyCarrierImage .\photo1_key.bmp -DataCarrierImage .\photo2_data.bmp

Key Features

  • Crypto-secure RNG: Uses RandomNumberGenerator (PS7/.NET 5+) or RNGCryptoServiceProvider (PS5.1)
  • AES-256-GCM encryption: Export passphrases with cryptographically secure keys
  • LSB steganography: Hide encrypted data inside BMP/PNG/JPEG images — send via separate channels
  • Offline password sharing: Two stego images containing key + encrypted data — neither image is sufficient alone
  • Auditable reconstruction: Save rolls as JSON; reconstruct exact passphrase later
  • Enterprise entropy: Meets Tier 0 (≥80 bits) requirements out of the box

Use Case: Offline Password Sharing

This module was designed to solve a real problem: sharing passwords securely with no network dependency.

Alice generates a password, then:

  1. Exports it encrypted to two separate stego images (e.g., office.jpg → office_key.bmp, team.jpg → team_data.bmp)
  2. Sends office_key.bmp via one channel (email, USB stick, hand-delivery)
  3. Sends team_data.bmp via a separate channel (Slack, different USB, different person)
  4. Bob receives both images, then: Get-Password -Import -KeyCarrierImage office_key.bmp -DataCarrierImage team_data.bmp
  5. The password is recovered without ever existing in plaintext on any network

Module State: $global:DICEWARE

Every call to New-Password automatically stores the result in $global:DICEWARE, a global variable that holds:

  • Password — SecureString (never plain text in memory by default)
  • Rolls — Array of 5-digit word roll keys
  • SaltRolls — Array of 3-digit salt character rolls (if applicable)
  • SaltPosition — Where the salt was inserted
  • Separator, UppercaseFirstLetter — Reconstruction settings
  • Entropy — Detailed entropy breakdown (word bits, salt bits, position bits, total)

Why this matters: You can generate a password, use it immediately, and then decide later to export or hide it without regenerating:

New-Password  → [use it]  → Save-PasswordToFile or Hide-PasswordFiles

Or retrieve it: Get-LastPassword or Get-LastPassword -CopyToClipboard

Available Functions

New-Password

Generate a diceware passphrase with options for word count, separators, salt, and encryption

Get-Password

Import from encrypted files or stego images; reconstruct from saved rolls

New-DicewareRandomString

Generate random special chars/digits using the 3-dice table

Save-PasswordToFile

Export passphrase as AES-256-GCM encrypted JSON + key file

Get-PasswordFromFile

Decrypt and restore from .key + .jsonenc files

Hide-PasswordFiles

Embed encrypted password data inside stego BMP images

  • Enhanced stego support: Improved steganography handling with two-channel password sharing (key image + data image sent separately)
  • Security improvements: AES-256-GCM encryption, better RNG handling across PS5.1 and PS7, enterprise entropy targets (Tier 0 ≥ 80 bits)

  • Auditable design: JSON export/import of rolls, reconstruction commands, and complete offline workflows
